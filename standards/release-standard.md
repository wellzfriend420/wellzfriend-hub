# リリース標準

> 状態: approved / 管理責任: リリース管理責任者 / 最終確認日: 2026-07-04

1. 対象範囲、受入条件、Version、責任役割を確定する。
2. 機能、回帰、安全性、プライバシー、運用性をリスクに応じて検証する。
3. 変更履歴、リリースノート、運用・復旧文書を更新する。
4. [リリース確認表](../templates/release-checklist.md)で証跡を確認し、権限を持つ役割が承認する。
5. 段階的に提供し、正常性と利用者影響を監視する。
6. 異常時は事前定義した停止・復旧条件を適用する。
7. 完了後に結果と学びを台帳・標準へ反映する。

緊急リリースでも検証、承認、追跡、復旧を省略しません。特定ストアやGitHubの操作は各プラットフォームガイドに記載します。
