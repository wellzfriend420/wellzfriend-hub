# リスク管理

アプリ事業横断のリスクと対応状況を管理する正本です。[risk-register.yaml](risk-register.yaml)をリスク管理責任者が月次、重大変化時は即時更新します。

可能性と影響は `low`, `medium`, `high`, `critical`、状態は `identified`, `treating`, `monitoring`, `accepted`, `closed` とします。各リスクに対象ID、予防、検知、対応、責任役割、次回確認日を持たせます。受容には承認役割と根拠が必要です。

攻撃手順、脆弱性の悪用詳細、顧客情報、資格情報、秘密情報は保存しません。サンプルは実在しません。
