# リリースとタグ

- [Version管理標準](../../../standards/version-management-standard.md)に従い、Gitタグは `vX.Y.Z` を基本とする。
- 公開Versionに対応するタグは移動・再利用しない。
- GitHub Releaseには[リリースノート雛形](../../../templates/release-notes.md)を用い、変更、影響、移行、既知の問題を記載する。
- 公開前に[リリース確認表](../../../templates/release-checklist.md)を完了する。
- 緊急修正でも検証、承認、変更履歴、復旧手順を省略しない。
